## What you need to install

<AnnotatedCode
  title="Global installs — run once per machine"
  language="bash"
  code={`# 1. Install Node 22 LTS (use nvm or the official installer)
nvm install 22
nvm use 22
node --version   # should print v22.x.x

# 2. Install global tools
npm install --global @rushstack/heft yo @microsoft/generator-sharepoint

# 3. Verify
yo --version
heft --version`}
  annotations={[
    { line: 2, label: 'Node 22 LTS', explanation: 'SPFx 1.22 requires Node 18 or 22 LTS. Node 20 also works but 22 is preferred for longer support. Do NOT use odd-numbered versions (Node 19, 21, 23) — they are not LTS and SPFx does not test against them.' },
    { line: 7, label: '@rushstack/heft', explanation: 'Heft is the build orchestrator that replaces gulp. It runs TypeScript compilation, Jest tests, and solution packaging through a single CLI.' },
    { line: 7, label: '@microsoft/generator-sharepoint', explanation: 'The Yeoman generator scaffolds new SPFx projects. It creates the correct folder structure, config files, and a starter web part.' },
  ]}
/>

<AnalogyBridge

<DangerZone
  title="Do NOT install gulp globally for SPFx 1.22+"
  wrong={`npm install --global gulp
gulp serve   # This was the old workflow — it no longer works`}
  right={`heft start   # Local dev server (replaces gulp serve)
heft build   # Production build (replaces gulp build)`}
  explanation="If you follow an older tutorial that says 'npm install -g gulp' and 'gulp serve', stop. As of SPFx 1.22, the Heft toolchain fully replaces gulp. Installing gulp globally can cause version conflicts."
/>
